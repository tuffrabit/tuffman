# tuffman PRD

## Goals

tuffman should be a, cli for now, AI agent orchestrator. tuffman will manage multiple AI agents across simple and complex tasks. For now tuffman will be focused on programming/coding tasks. tuffman should be able to explore code bases, write new software features, fix bugs, refactor implementations, and write documentation.

## Implementation

### First Principles

- model configuration
- local model loading with paramter control
- multiple API support (open ai, anthropic, lm studio, ollama, etc)

### High Order Features

- local model first design
- tool calling
- mcp support
- multi agent orchestration
- context management
- semi-formal reasoning

## Loose Ideas

- tuffman mascot
- targeted sub agent spawning
- code base exploration tool, perhaps with an indexing feature
- tui instead of raw cli
